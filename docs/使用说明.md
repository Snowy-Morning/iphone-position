# iPhone 虚拟定位 — 使用说明

通过 USB 修改 iPhone **系统级 GPS**，使「查找」、地图、位置共享显示虚拟坐标。

> **独立发布包，无需安装 Python。**

> **请解压到纯英文路径**（如 `D:\PositionTool`），不要放在中文目录下。

> 需保持 USB 连接；断开或关闭工具后恢复真实定位。

---

## 发布包内容

解压后应包含：`PositionTool.exe`、`_internal\`、`runtime\`、若干 `.bat` 文件。  
缺任何一项请重新解压完整 zip。

---

## 首次配置（按顺序）

| 步骤 | 操作 |
|------|------|
| 1 | 双击 `install_apple_usb.bat` 安装 Apple USB 服务 |
| 2 | USB 连接 iPhone，解锁，点「信任此电脑」 |
| 3 | 双击 `download_ddi.bat` 下载开发者镜像 |
| 4 | 双击 `enable_developer_mode.bat`，手机开启**开发者模式**并重启 |
| 5 | 运行 `check_env.bat`，确认 USB、隧道、DDI 均为 OK |

> `check_env` 显示开发者模式 `false` 时，需完成第 4 步后才能应用定位。

---

## 每次使用

```
1. USB 连接 iPhone 并解锁
2. 右键 start_tunnel.bat → 以管理员运行（保持窗口打开）
3. 双击 start_app.bat
4. 选点后点「应用定位」
5. 用完点「恢复真实定位」
```

---

## 自定义快捷地点

编辑同目录 `config.py` 中的 `PRESETS`，保存后重启 `start_app.bat`。

---

## 常见问题

| 现象 | 处理 |
|------|------|
| `start_app.bat` 一闪而过 | 解压到英文路径；窗口会显示 `crash.log` 内容 |
| 找不到 PositionTool.exe | 不要只复制 bat，需完整解压 |
| 找不到设备 | `check_env.bat`，检查 USB、信任、隧道 |
| 隧道未建立 | 先插手机再开 `start_tunnel.bat`；需管理员权限 |
| 开发者模式未开 | `enable_developer_mode.bat` |
| DDI 失败 | `download_ddi.bat`，国内可开 VPN |
| 查找不更新 | 完全退出「查找」重开，等 1–3 分钟 |

---

## 文件说明

| 文件 | 说明 |
|------|------|
| `start_app.bat` | 启动工具（推荐入口） |
| `start_tunnel.bat` | iOS 17+ USB 隧道 |
| `check_env.bat` | 一键环境检测 |
| `config.py` | 快捷地点配置 |
| `crash.log` | 启动失败时自动生成，便于排查 |
